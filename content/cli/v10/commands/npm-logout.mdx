### 개요

```bash
npm logout
```

참고: 이 명령은 작업 공간을 인식하지 않습니다.

### 설명

토큰 기반 인증을 지원하는 레지스트리에 로그인한 경우, 서버에 이 토큰의 세션을 종료하도록 알립니다. 이는 현재 환경뿐만 아니라 해당 토큰을 사용하는 모든 곳에서 토큰을 무효화합니다.

사용자 이름과 비밀번호 인증을 사용하는 레거시 레지스트리에 로그인한 경우, 사용자 구성에서 자격 증명을 지웁니다. 이 경우에는 _현재 환경에만_ 영향을 미칩니다.

`--scope`가 제공되면, 해당 스코프와 연결된 레지스트리의 자격 증명을 찾습니다(설정된 경우).

### 구성

#### `registry`

- 기본값: "https://registry.npmjs.org/"
- 유형: URL

npm 레지스트리의 기본 URL입니다.

#### `scope`

- 기본값: 현재 프로젝트의 스코프(있다면) 또는 ""
- 유형: String

스코프가 지정된 레지스트리에 대한 작업을 스코프와 연결합니다.

프라이빗 레지스트리에 로그인하거나 로그아웃할 때 유용합니다:

```
# 로그인하고 스코프를 사용자 정의 레지스트리에 연결
npm login --scope=@mycorp --registry=https://registry.mycorp.com

# 로그아웃하고 링크와 인증 토큰 제거
npm logout --scope=@mycorp
```

이렇게 하면 향후 `@mycorp/package` 패턴에 따라 지정된 패키지를 설치할 때 `@mycorp`가 레지스트리에 매핑됩니다.

또한 `npm init`이 스코프가 지정된 패키지를 생성하게 됩니다.

```
# 모든 기본값을 수락하고 "whatever" 대신 "@foo/whatever"라는 이름의 패키지 생성
npm init --scope=@foo --yes
```
