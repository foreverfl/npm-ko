### 설명

`npm install`과 같은 명령어와 `package.json`의 종속성 섹션에서는 패키지 이름 지정자를 사용합니다. 이는 "패키지"를 참조하는 다양한 형태일 수 있습니다. 예를 들어 패키지 이름, git URL, tarball, 또는 로컬 디렉토리 등이 있습니다. 이들은 일반적으로 이 패키지 이름 지정자를 사용하는 npm 명령어의 도움말 출력에서 `<package-spec>`으로 표시됩니다.

### 패키지 이름

- `[<@scope>/]<pkg>`
- `[<@scope>/]<pkg>@<tag>`
- `[<@scope>/]<pkg>@<version>`
- `[<@scope>/]<pkg>@<version range>`

범위가 있거나 없는 이름으로 패키지를 참조하며, 선택적으로 태그, 버전 또는 버전 범위를 포함할 수 있습니다. 이는 일반적으로 [registry](/cli/v10/using-npm/config#registry) 설정과 함께 사용되어 레지스트리의 패키지를 참조합니다.

예시:

- `npm`
- `@npmcli/arborist`
- `@npmcli/arborist@latest`
- `npm@6.13.1`
- `npm@^4.0.0`

### 별칭

- `<alias>@npm:<name>`

주로 `npm install` 같은 명령어와 `package.json`의 종속성 섹션에서 사용되며, 별칭으로 패키지를 참조합니다. `<alias>`는 `node_modules` 폴더에서 구체화되는 패키지의 이름이고, `<name>`은 구성된 레지스트리에서 찾을 수 있는 패키지 이름을 나타냅니다.

이름으로 패키지를 참조하는 방법에 대한 자세한 정보는 위의 '패키지 이름'을 참조하세요. 이름으로 패키지를 참조할 때 사용되는 레지스트리 구성에 대해서는 [registry](/cli/v10/using-npm/config#registry)를 참조하세요.

예시:

- `semver:@npm:@npmcli/semver-with-patch`
- `semver:@npm:semver@7.2.2`
- `semver:@npm:semver@legacy`

### 폴더

- `<folder>`

이는 로컬 파일 시스템의 패키지를 나타냅니다. 구체적으로 `package.json` 파일이 있는 폴더를 의미합니다. 혼동을 줄이기 위해 항상 `/` 또는 `./`(또는 귀하의 OS에 해당하는 것)로 시작해야 합니다. npm은 현재 하나 이상의 `/`가 포함된 문자열을 폴더로 해석하지만, 이는 레거시 동작으로 향후 버전에서 제거될 수 있습니다.

예시:

- `./my-package`
- `/opt/npm/my-package`

### Tarball

- `<tarball file>`
- `<tarball url>`

예시:

- `./my-package.tgz`
- `https://registry.npmjs.org/semver/-/semver-1.0.0.tgz`

로컬 파일 시스템이나 URL을 통해 원격으로 tarball 형식의 패키지를 나타냅니다. 이는 패키지가 레지스트리에 업로드될 때 존재하는 형식입니다.

### git URL

- `<git:// url>`
- `<github username>/<github project>`

git 저장소의 패키지를 나타냅니다. 이는 전체 git URL, git 축약형, 또는 GitHub의 사용자 이름/패키지일 수 있습니다. git 태그, 브랜치 또는 기타 git 참조를 `#ref`를 추가하여 지정할 수 있습니다.

예시:

- `https://github.com/npm/cli.git`
- `git@github.com:npm/cli.git`
- `git+ssh://git@github.com/npm/cli#v6.0.0`
- `github:npm/cli#HEAD`
- `npm/cli#c12ea07`
