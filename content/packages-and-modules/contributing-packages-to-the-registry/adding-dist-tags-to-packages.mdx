배포 태그(dist-tags)는 게시하는 패키지의 다양한 버전을 정리하고 라벨링하는 데 사용할 수 있는 사람이 읽을 수 있는 레이블입니다. dist-tags는 [시맨틱 버저닝][semver]을 보완합니다. 시맨틱 버전 번호보다 더 읽기 쉬울 뿐만 아니라, 태그를 사용하면 게시자가 패키지를 더 효과적으로 배포할 수 있습니다.

자세한 내용은 [`dist-tag` CLI 문서][dist-tag]를 참조하세요.

<Note>

**참고:** dist-tags는 시맨틱 버전과 네임스페이스를 공유하므로, 기존 버전 번호와 충돌하는 dist-tags는 피하세요. 숫자나 "v"로 시작하는 dist-tags를 사용하지 않는 것이 좋습니다.

</Note>

## dist-tag로 패키지 게시하기

기본적으로 `npm publish`를 실행하면 패키지에 `latest` dist-tag가 태그됩니다. 다른 dist-tag를 사용하려면 게시할 때 `--tag` 플래그를 사용하세요.

1. 명령줄에서 패키지의 루트 디렉토리로 이동합니다.

   ```
   cd /path/to/package
   ```

2. 다음 명령을 실행하되, `<tag>`를 사용하고자 하는 태그로 교체하세요:

   ```
   npm publish --tag <tag>
   ```

### 예시

"beta" dist-tag로 패키지를 게시하려면 패키지의 루트 디렉토리에서 명령줄에 다음 명령을 실행하세요:

```
npm publish --tag beta
```

## 패키지의 특정 버전에 dist-tag 추가하기

1. 명령줄에서 패키지의 루트 디렉토리로 이동합니다.

   ```
   cd /path/to/package
   ```

2. 다음 명령을 실행하되, `<package_name>`을 패키지 이름으로, `<version>`을 패키지 버전 번호로, `<tag>`를 배포 태그로 교체하세요:

   ```
   npm dist-tag add <package-name>@<version> [<tag>]
   ```

### 예시

"example-package" 패키지의 1.4.0 버전에 "stable" 태그를 추가하려면 다음 명령을 실행합니다:

```
npm dist-tag add example-package@1.4.0 stable
```
