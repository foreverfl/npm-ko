JavaScript 생태계를 건강하고, 신뢰할 수 있으며, 안전하게 유지하기 위해 자신이 소유한 npm 패키지에 중요한 업데이트를 할 때마다, [시맨틱 버저닝 규격][semver-org]을 따르는 새로운 버전 번호로 [`package.json` 파일][pkg-json]을 업데이트하여 패키지의 새 버전을 게시하는 것이 좋습니다. 시맨틱 버저닝 규격을 따르면, 귀하의 코드에 의존하는 다른 개발자가 해당 버전에서 변경된 내용의 범위를 이해하고 필요에 따라 자신의 코드를 조정할 수 있습니다.

<Note>

**참고:** 패키지 의존성을 깨뜨리는 변경을 도입하는 경우, 버전 **주 버전**을 증가시키는 것을 강력히 권장합니다. 자세한 내용은 아래를 참조하세요.

</Note>

## 게시된 패키지에서 시맨틱 버전 증가시키기

귀하의 코드에 의존하는 개발자를 돕기 위해 패키지 버전을 `1.0.0`부터 시작하고 다음과 같이 증가시키는 것이 좋습니다:

| 코드 상태               | 단계          | 규칙                                                         | 예시 버전 |
| ----------------------- | ------------- | ------------------------------------------------------------ | --------- |
| 첫 번째 릴리스          | 새로운 제품   | 1.0.0으로 시작                                               | 1.0.0     |
| 하위 호환 버그 수정     | 패치 릴리스   | 세 번째 숫자를 증가                                          | 1.0.1     |
| 하위 호환 새로운 기능   | 마이너 릴리스 | 가운데 숫자를 증가시키고 마지막 숫자를 0으로 설정            | 1.1.0     |
| 하위 호환성을 깨는 변경 | 메이저 릴리스 | 첫 번째 숫자를 증가시키고 가운데 및 마지막 숫자를 0으로 설정 | 2.0.0     |

## 시맨틱 버저닝을 사용하여 패키지에서 허용할 업데이트 유형 지정하기

패키지의 `package.json` 파일에서 의존성으로부터 허용할 업데이트 유형을 지정할 수 있습니다.

예를 들어, 1.0.4까지의 허용 가능한 버전 범위를 지정하려면 다음과 같은 구문을 사용하세요:

- 패치 릴리스: `1.0` 또는 `1.0.x` 또는 `~1.0.4`
- 마이너 릴리스: `1` 또는 `1.x` 또는 `^1.0.4`
- 메이저 릴리스: `*` 또는 `x`

시맨틱 버저닝 구문에 대한 자세한 내용은 [npm semver 계산기][semver-calc]를 참조하세요.

### 예시

```json
"dependencies": {
  "my_dep": "^1.0.0",
  "another_dep": "~2.2.0"
},
```
