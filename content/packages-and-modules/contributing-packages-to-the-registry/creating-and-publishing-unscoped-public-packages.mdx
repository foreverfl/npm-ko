---
title: 스코프 없는 공개 패키지 생성 및 게시
---

npm 사용자로서, 자신의 프로젝트에서 사용할 스코프가 없는 패키지를 만들고 다른 사람들이 자신의 프로젝트에서 사용할 수 있도록 npm 공개 레지스트리에 게시할 수 있습니다. 스코프가 없는 패키지는 항상 공개이며 패키지 이름으로만 참조됩니다:

```
package-name
```

패키지 범위, 접근 수준 및 가시성에 대한 자세한 내용은 "[패키지 범위, 접근 수준 및 가시성][pkg-viz]"을 참조하세요.

<Note>

**참고:** 공개 스코프가 없는 npm 패키지를 게시하기 전에 npm 사용자 계정에 [가입](https://www.npmjs.com/signup)해야 합니다.

</Note>

## 스코프가 없는 공개 패키지 생성하기

1. 명령줄에서 패키지를 위한 디렉토리를 생성합니다:

   ```
   mkdir my-test-package
   ```

2. 패키지의 루트 디렉토리로 이동합니다:

   ```
   cd my-test-package
   ```

3. git을 사용하여 패키지 코드를 관리하는 경우, 패키지 루트 디렉토리에서 다음 명령을 실행하되 `git-remote-url`을 패키지의 git 원격 URL로 교체하세요:

   ```
   git init
   git remote add origin git://git-remote-url
   ```

4. 패키지 루트 디렉토리에서 `npm init` 명령을 실행합니다.
5. 프롬프트에 응답하여 [`package.json`](https://docs.npmjs.com/about-package-json-and-package-lock-json-files) 파일을 생성합니다. 패키지 이름 지정에 대한 도움말은 "[패키지 이름 지침][pkg-name]"을 참조하세요.
6. 패키지 코드의 내용과 사용 방법을 설명하는 [README 파일][readme-file]을 만듭니다.
7. 선호하는 텍스트 편집기에서 패키지의 코드를 작성합니다.

## 민감하거나 불필요한 정보에 대한 패키지 내용 검토

레지스트리에 민감한 정보를 게시하면 사용자에게 해를 끼치고, 개발 인프라를 손상시키며, 수정하는 데 비용이 많이 들고, 법적 조치의 위험에 처할 수 있습니다. **레지스트리에 패키지를 게시하기 전에 개인 키, 비밀번호, [개인 식별 정보][pii](PII) 및 신용 카드 데이터와 같은 민감한 정보를 제거할 것을 강력히 권장합니다.**

테스트 데이터와 같은 덜 민감한 정보의 경우, `.npmignore` 또는 `.gitignore` 파일을 사용하여 레지스트리에 게시되는 것을 방지하세요. 자세한 내용은 [이 문서][developers]를 참조하세요.

## 패키지 테스트하기

버그 게시 가능성을 줄이기 위해 npm 레지스트리에 게시하기 전에 패키지를 테스트하는 것이 좋습니다. 패키지를 테스트하려면 패키지 디렉토리의 전체 경로로 `npm install`을 실행하세요:

```
npm install path/to/my-package
```

## 스코프가 없는 공개 패키지 게시하기

1. 명령줄에서 패키지의 루트 디렉토리로 이동합니다.

   ```
   cd /path/to/package
   ```

2. npm 레지스트리에 공개 패키지를 게시하려면 다음을 실행하세요:

   ```
   npm publish
   ```

   <Note>

   **참고:** GitHub Actions를 사용하여 패키지를 게시하는 경우 게시하는 각 패키지에 대한 출처 정보를 생성할 수 있습니다. 자세한 내용은 "[출처 명세서 생성하기][provenance-how-to]"를 참조하세요.

   </Note>

3. 공개 패키지 페이지를 보려면 `https://npmjs.com/package/*package-name*`을 방문하세요. `*package-name*`을 패키지 이름으로 교체하세요. 공개 패키지는 npm 웹사이트의 패키지 이름 아래에 `public`이라고 표시됩니다.

`publish` 명령에 대한 자세한 내용은 [CLI 문서][cli-publish]를 참조하세요.
