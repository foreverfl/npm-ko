---
title: 강력한 비밀번호 만들기
---

비밀번호 관리자를 사용하여 강력하고 고유한 비밀번호로 npm 계정을 보호하세요.

npm 계정을 위해 다음 조건을 충족하는 비밀번호를 선택하거나 생성해야 합니다:

- 10자 이상이어야 함
- 사용자 이름과 일치하거나 상당 부분 포함하지 않아야 함. 예를 들어 'username123'과 같은 비밀번호는 사용하지 마세요
- [Have I Been Pwned](https://haveibeenpwned.com/) 유출 데이터베이스에 알려진 적이 없는 비밀번호여야 함

계정의 보안을 유지하기 위해 다음과 같은 모범 사례를 따르는 것이 좋습니다:

- [1Password](https://1password.com/)와 같은 비밀번호 관리자를 사용하여 16자 이상의 비밀번호를 생성하세요.
- npm을 위한 고유한 비밀번호를 생성하세요. npm 비밀번호를 다른 곳에서 사용하고 해당 서비스가 침해당하면 공격자나 악의적인 행위자가 그 정보를 사용하여 npm 계정에 접근할 수 있습니다.
- 계정에 2단계 인증을 구성하세요. 자세한 내용은 "2단계 인증 정보"를 참조하세요.
- 잠재적인 협력자와도 절대 비밀번호를 공유하지 마세요. npm에서는 각 사람이 자신의 개인 계정을 사용해야 합니다. 협업 방법에 대한 자세한 내용은 "[npm 조직](/organizations)"을 참조하세요.

로그인, 계정 생성 또는 비밀번호 변경을 위해 비밀번호를 입력할 때 npm은 HaveIBeenPwned와 같은 데이터셋에 따라 입력한 비밀번호가 약한 것으로 간주되는지 확인합니다. 이전에 해당 비밀번호를 사용한 적이 없더라도 약한 것으로 식별될 수 있습니다.

npm은 비밀번호를 입력할 때만 검사하며, 입력한 비밀번호를 평문으로 저장하지 않습니다. 자세한 내용은 [HaveIBeenPwned](https://haveibeenpwned.com/)를 참조하세요.
