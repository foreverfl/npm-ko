계정에 2단계 인증이 활성화되어 있고 2FA 기기에 대한 접근 권한을 잃어버린 경우, 다음 방법을 사용하여 계정을 복구할 수 있습니다.

## 2단계 인증 기기를 분실한 경우

2단계 인증을 제공하는 기기를 분실한 경우, [2FA를 활성화][setup-recovery-codes]할 때 생성된 복구 코드를 사용하여 계정에 접근할 수 있습니다.

### 웹에서 복구 코드 사용하기

1. 저장해 둔 생성된 복구 코드를 찾습니다.

2. <>{shared['user-login'].text}</>

   <>{shared['user-login'].image}</>

3. <>{shared['use-recovery-code'].text}</>

   <>{shared['use-recovery-code'].image}</>

   _참고: TOTP 사용을 구성한 경우 대신 TOTP 프롬프트가 표시됩니다_

4. "복구 코드 사용" 프롬프트에 사용하지 않은 복구 코드를 입력합니다.

   <Screenshot src="/getting-started/setting-up-your-npm-user-account/user-a-recovery-code.png" alt="복구 코드를 입력할 수 있는 입력 상자가 있는 복구 코드 사용 프롬프트를 보여주는 스크린샷" />

5. 이제 npm에 로그인되었습니다.

6. (선택 사항) 2FA를 비활성화하려면 "[2FA 비활성화하기][removing-2fa-on-the-web]"를 참조하세요.

### 명령줄에서 복구 코드 사용하기

1. 계정에서 2FA를 활성화할 때 생성된 복구 코드를 찾습니다.

2. 명령줄에서 로그아웃한 경우 사용자 이름과 npm 비밀번호로 `npm login` 명령을 사용하여 로그인합니다.

3. 다음 프롬프트가 표시되면 사용하지 않은 복구 코드를 입력합니다:

   <Prompt>Enter one-time password:</Prompt>

4. 로그인하면 아래 명령을 사용하고 메시지가 표시되면 npm 비밀번호를 입력합니다.

   ```
   npm profile disable-2fa
   ```

5. 다음 프롬프트가 표시되면 또 다른 사용하지 않은 복구 코드를 입력합니다:

   <Prompt>Enter one-time password:</Prompt>

6. npm은 2단계 인증이 비활성화되었음을 확인합니다.

7. "[2단계 인증 구성하기][configuring-two-factor-authentication]"에 설명된 단계에 따라 2FA를 다시 활성화하고 새 복구 코드를 생성합니다.

<Note>

**참고:** 복구 코드를 사용하여 2FA를 다시 활성화하면 동일한 npm 계정 이름으로 새 인증기 계정이 생성될 수 있습니다.

[시간 기반 일회용 비밀번호(TOTP)][totp] 모바일 앱을 사용 중이고 이전 인증기 계정을 삭제하려면 해당 인증기의 단계를 따르세요.

</Note>

## 복구 코드 보기 및 재생성하기

<Note>

**참고:** 새 코드 세트를 생성하면 이전의 모든 복구 코드가 무효화됩니다. 각 코드는 한 번만 사용할 수 있습니다.

</Note>

1. <>{shared['user-login'].text}</>

   <>{shared['user-login'].image}</>

2. <>{shared['account-settings'].text}</>

   <>{shared['account-settings'].image}</>

3. 계정 설정 페이지의 "2단계 인증" 아래에서 **2FA 수정**을 클릭합니다.

   <Screenshot src="/getting-started/setting-up-your-npm-user-account/2fa-modify.png" alt="2FA 수정 버튼을 보여주는 스크린샷" />

4. "복구 코드 관리"를 클릭하여 복구 코드를 확인합니다.

   <Screenshot src="/getting-started/setting-up-your-npm-user-account/view-recovery-codes.png" alt="기존 복구 코드와 새 복구 코드 세트를 생성하는 버튼을 보여주는 스크린샷" />

5. "코드 재생성"을 클릭하여 새 코드 세트를 생성합니다.

## 복구 코드를 분실한 경우

2FA 기기와 복구 코드를 모두 분실한 경우, 지원 팀에 연락하여 계정 복구를 시도할 수 있습니다. 요청을 더 빨리 처리할 수 있도록 가능한 한 많은 정보를 제공하세요.

1. <>{shared['user-login'].text}</>

   <>{shared['user-login'].image}</>

2. <>{shared['use-recovery-code'].text}</>

   <>{shared['use-recovery-code'].image}</>

3. "복구 코드 사용" 양식 아래에서 **계정 복구 시도하기**를 클릭합니다.

4. <>{shared['start-account-recovery'].text}</>

   <>{shared['start-account-recovery'].image}</>

5. 등록된 이메일에 접근할 수 있는 경우, 이메일로 전송된 일회용 비밀번호를 **일회용 비밀번호** 필드에 입력한 다음 **이메일 주소 확인**을 클릭합니다. _등록된 이메일에 접근할 수 없는 경우, 양식 하단에서 **이메일 확인 건너뛰기**를 선택하세요._

6. <>{shared['support-ticket-form'].text}</>

   - <>{shared['enter-email-address'].text}</>
   - **어떻게 도와드릴까요?** 섹션에서 **2단계 인증(2FA) 재설정**을 선택합니다.
   - <>{shared['support-ticket-other'].text}</><>{shared['connect-to-accounts'].text}</>

   <Screenshot src="/getting-started/setting-up-your-npm-user-account/recover-account.png" alt="기존 복구 코드와 새 복구 코드 세트를 생성하는 버튼을 보여주는 스크린샷" />

7. <>{shared['submit-support-ticket'].text}</>
